# Governance

## Project Model

`retrievalx` is maintained by the `@retrievalx/maintainers` team.

The project uses a maintainer-led governance model with transparent technical
decision making in issues and pull requests.

## Roles

### Maintainers

- Set roadmap and release policy
- Review and merge pull requests
- Triage issues and security reports
- Enforce quality and conduct standards

### Contributors

- Propose and implement improvements
- Add tests and documentation with code changes
- Follow project standards in [CONTRIBUTING.md](CONTRIBUTING.md)

## Decision Process

- Small changes: decided in PR review by maintainers.
- Significant changes: discussed in issues/PRs; maintainers decide after
  feedback and technical review.
- Security decisions follow [SECURITY.md](SECURITY.md).

## Release Authority

Only maintainers may publish releases and PyPI artifacts.
Release workflow changes require maintainer approval.

## Conflict Resolution

If consensus is unclear, maintainers make a final decision based on:

- correctness and safety
- backward compatibility
- long-term maintainability
- performance impact

## Becoming a Maintainer

Maintainer invitations are based on sustained high-quality contributions:

- repeated technically sound PRs
- thoughtful reviews
- strong reliability and communication
- stewardship of users and contributors
